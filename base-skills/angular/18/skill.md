# Angular 18 — Skill File

## What changed in this version

The Signals API (`signal`, `computed`, `effect`) is now stable and is the preferred reactivity primitive — replacing `BehaviorSubject` patterns for local component state. The new built-in control flow syntax (`@if`, `@for`, `@switch`) is stable and replaces structural directives (`*ngIf`, `*ngFor`). `@defer` blocks for lazy-loading template sections are also stable. Zoneless change detection is available experimentally via `provideExperimentalZoneless()`, making `zone.js` optional. Signal-based component inputs (`input()`, `output()`, `model()`) are stable and replace `@Input()`/`@Output()` decorators.

---

## Core patterns — DO

**Use signal-based inputs and outputs**
```typescript
// component.ts
readonly count = input<number>(0);          // replaces @Input()
readonly increment = output<number>();       // replaces @Output() EventEmitter
readonly value = model<string>('');         // two-way bindable signal
```
Signal inputs are lazy, typed, and composable with `computed()` — they do not require `ngOnChanges`.

**Derive state with `computed()`**
```typescript
readonly doubled = computed(() => this.count() * 2);
```
`computed()` memoizes automatically; never derive values inside a template expression when a `computed()` will do.

**Use built-in control flow syntax in templates**
```html
@if (user()) {
  <p>{{ user()!.name }}</p>
} @else {
  <p>Loading...</p>
}

@for (item of items(); track item.id) {
  <li>{{ item.label }}</li>
}
```
`track` is mandatory in `@for` — use a unique identifier, never `$index` unless items have no stable id.

**Lazy-load heavy template sections with `@defer`**
```html
@defer (on viewport) {
  <app-heavy-chart />
} @placeholder {
  <div>Chart placeholder</div>
}
```
`@defer` is the canonical way to lazy-load components; do not use dynamic `import()` + manual component factories.

**Clean up subscriptions with `takeUntilDestroyed()`**
```typescript
this.data$.pipe(takeUntilDestroyed()).subscribe(...);
```
`takeUntilDestroyed()` (from `@angular/core/rxjs-interop`) replaces manual `Subject`/`takeUntil` teardown patterns.

**Bridge RxJS and Signals at boundaries**
```typescript
readonly items = toSignal(this.items$, { initialValue: [] });
```
Use `toSignal` / `toObservable` from `@angular/core/rxjs-interop` at the seam between Observable streams and signal-based components.

## Anti-patterns — AVOID in 18

**`*ngIf` / `*ngFor` / `*ngSwitch` structural directives** — These still work but are deprecated in favour of `@if`, `@for`, `@switch`. Never generate new code using `*ngIf` or `*ngFor`.

**`@Input()` / `@Output()` decorators for new components** — The signal-based `input()` / `output()` / `model()` API is now stable. Decorator-based I/O is legacy for new code.

**Manual `ngOnDestroy` + `Subject` for unsubscribing** — Replace with `takeUntilDestroyed()`. The old pattern is verbose and error-prone.

**`BehaviorSubject` for local component state** — Use `signal()` for synchronous local state. Reserve `BehaviorSubject` only for shared service streams that must interop with RxJS pipelines.

**`CommonModule` imports in standalone components** — In Angular 18 standalone components, import only what is needed (`NgClass`, `AsyncPipe`, etc.) or nothing if using built-in control flow, which requires no import.

## Version-specific notes for AI agents

- `standalone: true` is the default for new components generated by the CLI in v18. Never emit `NgModule`-based components unless the project explicitly uses modules.
- `@for` requires a `track` expression — emitting `@for` without `track` is a compile error.
- `effect()` must be called inside an injection context (constructor or `runInInjectionContext`); calling it in `ngOnInit` without `runInInjectionContext` throws at runtime.
- `provideExperimentalZoneless()` and `provideZoneChangeDetection()` are mutually exclusive in `app.config.ts` — never emit both.
- `inject()` is the preferred way to get dependencies inside functions and `computed()`; do not use constructor injection inside standalone functions or signal factories.
